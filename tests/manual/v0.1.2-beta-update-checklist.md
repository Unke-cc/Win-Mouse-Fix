# Win Mouse Fix 0.1.2 Beta 更新检查清单（历史记录）

> 本文件仅记录 0.1.2 Beta 的发布检查，不代表当前 `0.1.3 Beta` 的完整检查要求。当前版本请使用 [0.1.3 Beta 更新检查清单](./v0.1.3-beta-update-checklist.md)。

本清单覆盖本轮新增和调整的功能。记录 Windows 版本、鼠标型号、安装方式、测试日期，并为失败项目保留截图或录屏。

## 1. 按键功能列表

- [ ] 分别录入点击、双击、长按、按住并滚动、按住并拖动。
- [ ] 确认普通操作完整显示通用动作；按住并滚动只显示滚动融合动作，按住并拖动只显示拖动融合动作。
- [ ] 确认滚动融合项包含缩放、音量调节、标签页切换、页面前进后退、虚拟桌面切换、桌面与开始菜单，不包含快速滚动。
- [ ] 确认拖动融合项包含滚动与移动、虚拟桌面与任务视图、网页与标签导航。
- [ ] 将缩放或音量调节设为按住并滚动动作，连续滚动时确认每格按上下方向执行。
- [ ] 测试网页与标签导航的四个方向，确认向左后退、向右前进、向上切换上一个标签页、向下切换下一个标签页。
- [ ] 分别测试“先滚动、后长按”和“先长按、后滚动”，确认仅最先成立的操作执行。
- [ ] 确认普通操作列表中“执行原功能”位于第一项、“不执行动作”位于最后一项；两个融合动作列表不包含普通单次动作。
- [ ] 新建两个以上虚拟桌面，确认“虚拟桌面与任务视图”左右拖动可以连续往返切换，每次切换后所有按钮映射立即可用。
- [ ] 开启“全屏时自动暂停”，确认空桌面、任务视图和普通最大化窗口不会暂停，真正的全屏视频或游戏仍会暂停。
- [ ] 首次启动分别确认三键鼠标获得 4 条中键默认值，五键以上鼠标获得按键 4/5 的 6 条默认值。
- [ ] 点击“恢复默认值...”，确认可选择三键鼠标或五键及多键鼠标。
- [ ] 在滚动页确认 `Shift`、`Alt`、`Win`、`Ctrl` 默认对应水平、快速、精确、缩放，并可改为其他不重复的修饰键或关闭。
- [ ] 修改任意映射后退出并重新启动，确认设置保持不变。

## 2. 页面与滚动条

- [ ] 切换通用、滚动和关于页，确认页面外层没有滚动条。
- [ ] 在通用页添加多个排除应用，确认只在应用列表内部出现滚动条。
- [ ] 在按钮页增加多条映射，确认只在映射列表内部出现滚动条。
- [ ] 确认按钮页录入区域、说明文字和底部按钮不随列表滚动。
- [ ] 分别在 Windows 缩放比例 100%、125%、150% 下重复页面切换。

## 3. 正式版与 Beta 版更新

- [ ] 默认启动后确认“获取 Beta 版更新”处于关闭状态。
- [ ] 关闭 Beta 开关检查更新，确认只比较正式版本。
- [ ] 开启 Beta 开关检查更新，确认正式版和 Beta 版都参与比较。
- [ ] 退出并重新启动，确认 Beta 开关状态保持不变。
- [ ] GitHub 正常时确认显示 GitHub 来源，并打开对应 GitHub Release 页面。
- [ ] 让 GitHub 请求失败，确认程序自动改查 Gitee，并显示 Gitee 来源。
- [ ] GitHub 正常返回“没有更新”时，确认程序不再查询 Gitee。
- [ ] GitHub 和 Gitee 都不可用时，确认显示明确提示，窗口不退出。
- [ ] 首次进入关于页时，确认三秒内优先选择 GitHub；GitHub 不可用时自动选择 Gitee。
- [ ] 关于页中的项目主页、许可和检查更新打开的页面均属于同一来源。
- [ ] GitHub 与 Gitee 都不可用时，关于页持续显示不可用原因，项目主页和许可按钮不可点击。
- [ ] 确认 `0.1.2-beta.2` 高于 `0.1.2-beta.1`，且正式版 `0.1.2` 高于同版本 Beta。
- [ ] Gitee Release 的 `draft` 值为 `null` 时，确认仍能正常读取更新。
- [ ] 确认正式版没有标记为预发布，Beta 版已标记为预发布。
- [ ] 确认 GitHub 与 Gitee 使用相同 tag，并上传同一版本安装包。

## 4. 配置管理入口

- [ ] 通用页底部显示“配置管理…”按钮。
- [ ] 点击后打开独立配置管理窗口，关闭后返回通用页。
- [ ] 配置管理窗口不改变主窗口当前页面和位置。

## 5. 多套配置切换

- [ ] 新建配置副本，确认它复制当前按钮、滚轮、应用排除、启动设置和 Beta 开关。
- [ ] 修改新配置后切回原配置，确认两套设置互不影响。
- [ ] 退出并重新启动，确认继续使用最后选择的配置。
- [ ] 重命名配置，确认新名称立即显示并在重启后保留。
- [ ] 删除非当前配置，确认当前设置不变。
- [ ] 删除当前配置，确认自动切换到剩余配置。
- [ ] 尝试删除最后一个配置，确认程序拒绝删除并保留配置。
- [ ] 尝试使用空名称、路径字符或重复名称，确认程序给出明确提示。

## 6. 导入与导出

- [ ] 导出当前配置，确认生成可读取的 `.json` 文件。
- [ ] 使用新名称导入刚导出的文件，确认全部设置保持一致。
- [ ] 导入旧版 `configVersion: 1` 配置，确认完成转换且旧方向设置保留。
- [ ] 导入内容无效、版本不支持或扩展名错误的文件，确认当前配置不被替换。
- [ ] 导入配置后重启，确认导入的配置仍为当前配置。

## 7. 备份与恢复

- [ ] 点击“立即备份”，确认备份列表出现新的时间戳文件。
- [ ] 修改按钮、滚轮、应用排除和 Beta 开关后恢复备份，确认全部恢复。
- [ ] 导入配置前确认程序自动创建当前配置备份。
- [ ] 恢复旧备份前确认程序先备份当前配置。
- [ ] 连续创建多个备份，确认文件不会互相覆盖。
- [ ] 选择不存在或已损坏的备份，确认当前配置不被替换。

## 8. 配置保存位置

- [ ] 当前生效配置位于 `%AppData%\WinMouseFix\config.json`。
- [ ] 多套配置位于 `%AppData%\WinMouseFix\profiles`。
- [ ] 备份位于 `%AppData%\WinMouseFix\backups`。
- [ ] 当前配置名称记录在 `%AppData%\WinMouseFix\active-profile.txt`。
- [ ] 安装新版和卸载程序后，用户配置、配置档案和备份仍然保留。

## 9. 鼠标与系统检查

- [ ] 使用中键、按键 4、按键 5 分别测试全部主要操作。
- [ ] 在浏览器、文件管理器、普通窗口、全屏视频和全屏游戏中测试。
- [ ] 在排除应用和全屏暂停状态下切换配置，确认鼠标恢复系统原始行为。
- [ ] 在 Windows 10 22H2 x64 和 Windows 11 x64 上完成安装、更新、启动和卸载。
- [ ] 登录 Windows 后运行、托盘停止/启动、单实例和后台启动保持正常。

## 10. 发布文件

- [ ] `WinMouseFix.exe`、鼠标核心、默认配置和图标均包含在发布目录。
- [ ] 安装包版本、程序版本、Git tag、GitHub Release 和 Gitee Release 一致。
- [ ] 安装包可以覆盖旧版本安装，用户设置不丢失。
- [ ] GitHub 和 Gitee 下载页面均可打开并下载对应安装包。

## 结果

- [ ] 全部项目通过。
- [ ] 失败项目已有复现步骤和截图或录屏。
- [ ] 失败项目修复后已重新检查相关项目。
