# Win Mouse Fix 应用设计文档

## 1. 文档信息

- 状态：已实现并持续更新
- 适用版本：0.1.4 Beta
- 用途：统一产品范围、用户操作、程序组成、发布方式和验收标准

## 2. 产品目标

Win Mouse Fix 是一款基于 AutoHotkey v2 的 Windows 鼠标快捷编辑工具，参考 MacMouseFix 的使用方式，让用户通过可视化页面设置鼠标按键、滚轮和组合动作。

AutoHotkey v2 负责全部核心鼠标功能，GUI 负责把原本需要编写脚本的操作转换为选择项、开关、滑块和表单。目标用户不需要理解 AutoHotkey 语法，只需描述想要的鼠标行为并完成选择。

程序应满足以下要求：

- 用户无需单独安装 AutoHotkey；GUI 使用目标 Windows 系统中的 .NET Framework 4.8。
- 设置完成后立即生效，通常不需要手动重启。
- 指定应用运行时，可暂停部分或全部增强功能，恢复鼠标原始行为。
- 设置在程序关闭、更新或系统重启后仍能保留。
- 程序可常驻系统托盘，并清楚显示当前状态。
- 重复启动程序时只保留一个设置窗口、托盘图标和鼠标核心。
- GUI 只提供经过定义的鼠标功能，不要求用户直接编辑脚本。

本产品不是通用 AutoHotkey 编辑器。核心价值是降低鼠标配置难度，同时保留 AutoHotkey v2 在鼠标按键、滚轮、组合动作和应用规则方面的能力。

## 3. 0.1.4 Beta 已实现范围

当前版本包含：

- 中键、按键 4、按键 5 的点击、双击、长按、按住并滚动和按住并拖动。
- 普通操作完整动作列表，以及按住滚动和按住拖动专用的融合动作列表。
- 滚轮方向、速度、平滑度，以及水平、快速、精确和缩放四种滚轮功能。
- 仅允许 Ctrl、Alt、Shift、Win 的单键或多键组合；普通键和完整字符快捷键不接受。
- 三键鼠标与五键或多键鼠标默认值自动识别，以及“恢复默认值...”手动选择。
- 总开关、系统托盘、应用排除、全屏暂停、开机自动运行和单实例。
- 配置自动保存、多套配置切换、导入、导出、备份和恢复。
- GitHub 优先、Gitee 回退、正式版/Beta 更新检查。
- 触摸板高频滚轮输入保护、虚拟桌面切换和异常按键状态恢复。
- 设置窗口关闭 5 分钟后自动进入轻量模式；托盘可立即手动进入轻量模式，鼠标核心与配置继续运行。
- 安装版和便携版发布。

当前暂不包含云端同步、用户账号、在线配置分享、企业集中管理、任意脚本编辑、多设备独立配置，以及按设备来源精确区分触摸板和实体鼠标。

## 4. 用户使用流程

1. 用户安装或直接运行 Win Mouse Fix。
2. 用户主动启动时显示设置窗口；登录 Windows 后运行时使用 `--background` 静默进入托盘。
3. 用户设置鼠标按键、滚轮或应用排除规则。
4. 用户修改选项后，程序自动保存配置并立即更新行为。
5. 如果某项设置需要重新初始化，程序只重新启动鼠标功能部分，设置窗口保持打开。
6. 用户切换到排除列表中的应用时，增强功能自动暂停。
7. 用户离开该应用后，原有设置自动恢复。

## 5. 界面设计

主窗口采用顶部横向选择器，包含以下页面：

- **通用**：总开关、全屏暂停、应用排除、登录运行，以及底部的配置管理入口。
- **按钮**：中键、按键 4、按键 5、组合动作，以及三键或五键以上鼠标默认值。
- **滚动**：方向、速度、平滑度，以及水平、快速、精确和缩放四种滚轮动作；每项支持 Ctrl、Alt、Shift、Win 组合键。
- **关于**：版本、作者、许可状态、项目主页、Beta 更新开关，以及 GitHub/Gitee 更新检查。

通用、滚动和关于页不显示整页滚动条。按钮页只允许映射列表内部滚动；通用页的应用排除列表在内容较多时使用自己的内部滚动区域。

界面和托盘图标应区分正常启用、用户暂停、因当前应用暂停、配置重新载入中，以及配置读取失败后使用默认设置。普通选项不设置额外的“应用”按钮，修改后自动保存；需要重新初始化时只重新启动鼠标功能部分。

## 6. 技术与发布方案

### 6.1 产品结构

程序分为“托盘控制、可视化界面”和“AutoHotkey v2 鼠标核心”三部分。鼠标核心是固定基础，负责所有实际鼠标行为；托盘控制负责常驻和启动设置界面；设置界面负责让用户以简单方式编辑核心所需的配置。

GUI 确定使用 **.NET Framework 4.8 + WPF**，只在设置界面打开时运行。独立的 **.NET Framework 4.8 + Windows Forms** 托盘进程负责常驻托盘和核心进程控制，`System.Text.Json` 用于读写配置。

### 6.2 AutoHotkey v2 鼠标核心

AutoHotkey v2 是本项目确定使用的核心技术，负责鼠标按键、滚轮、组合动作、活动应用识别、应用排除和配置切换。核心读取固定格式的配置并执行预先提供的动作，不要求普通用户编写脚本。

Ahk2Exe 负责把鼠标核心及 AutoHotkey v2 运行环境打包为应用内部程序。用户安装 Win Mouse Fix 后无需单独安装 AutoHotkey。

### 6.3 GUI 封装

GUI 将核心能力显示为容易理解的控件，例如“侧键 1 执行什么动作”“是否反转滚轮”“哪些应用中暂停功能”。用户提交表单后，GUI 检查输入并生成核心能够读取的配置。

GUI 不直接实现鼠标行为，也不向普通用户展示脚本代码。新增鼠标功能时，必须同时增加核心动作、配置定义和对应的 GUI 控件。

GUI 使用 Windows 命名对象确保只运行一个实例。普通重复启动会显示已有设置窗口；重复 `--background` 启动会直接结束，不打断当前使用。登录运行登记使用 `"WinMouseFix.exe" --background`，因此登录后只显示托盘图标。

### 6.4 轻量模式

托盘服务和 `CoreProcessService` 由独立的 `WinMouseFix.Tray.exe` 持有，设置页面由 `WinMouseFix.exe` 持有。用户关闭设置窗口，或设置窗口连续 5 分钟没有鼠标和键盘操作后，设置进程退出；任何新的点击、滚动、移动或键盘输入都会重新计时。

5 分钟到期或用户从托盘选择“立即进入轻量模式”时，设置进程先保存当前配置并退出，只保留托盘进程和 AutoHotkey v2 核心。用户再次打开设置时重新启动 `WinMouseFix.exe` 并读取当前配置，不重复启动核心。

### 6.5 两部分通信

当前版本由 GUI 保存结构固定的 JSON 配置，AutoHotkey v2 核心监听文件变化并自动读取。核心检查间隔为 250 毫秒，GUI 保存使用 400 毫秒延迟，避免滑块连续变化时频繁写入。

Windows Named Pipes 保留为后续方案，用于应用配置、暂停、恢复、重新启动核心、查询状态和执行结果。

GUI 是配置文件的唯一修改者；AutoHotkey v2 只读取经过检查的配置并执行鼠标行为。配置包含版本号；若用户配置不能读取，核心使用默认配置继续运行。完整的结果确认和错误返回需要在 Named Pipes 完成后加入。

开关和选项在修改后立即发送，连续修改使用界面现有的延迟保存机制。大多数设置直接更新运行状态，只有不能直接更新的设置才重新启动 AutoHotkey v2 核心，GUI 和托盘始终保持运行。

### 6.6 Inno Setup

Inno Setup 将界面、鼠标功能及所需运行文件组合为统一安装程序。安装范围为当前用户，默认目录为 `%LocalAppData%\Programs\Win Mouse Fix`；安装器创建开始菜单入口，并提供可选桌面快捷方式。

固定 `AppId` 用于识别同一产品并支持直接更新。安装和卸载均不删除 `%AppData%\WinMouseFix`，因此用户设置会保留；卸载时删除 Win Mouse Fix 的登录运行登记。`.\scripts\package.ps1` 生成 `dist/WinMouseFix-Setup-0.1.4-beta.exe`。

### 6.7 系统与体积范围

正式支持 Windows 11 x64，兼容 Windows 10 22H2 x64；当前版本不提供 ARM64 和 32 位版本。GUI 主程序及其依赖在 `dist/app` 中合计不得超过 10 MiB，鼠标核心、配置和资源文件单独统计。

## 7. 程序组成

```text
src/
  gui/
    WinMouseFix.Gui.csproj
    App.xaml
    MainWindow.xaml
    Controls/
    Models/
    Services/
  tray/

  engine/

    lib/
      ActionManager.ahk

      MouseManager.ahk
  shared/
    config.schema.json
    messages.json
config/
  default.json
assets/
scripts/
tests/manual/
dist/
```

- `gui/`：.NET Framework 4.8 + WPF 设置应用，按需启动并展示设置页面。
- `tray/`：.NET Framework 4.8 + Windows Forms 托盘应用，常驻托盘并管理鼠标核心。
- `engine/MouseEngine.ahk`：启动 AutoHotkey v2 鼠标核心。
- `MouseManager`：接收并处理鼠标行为。
- `ActionManager`：把用户选择转换为具体动作。
- `AppRuleManager`：识别当前应用并决定是否暂停功能。
- `ConfigManager`：读取、检查、保存和更新配置。
- `shared/`：规定配置格式以及界面与鼠标功能之间的消息格式。

## 8. 配置设计

默认配置位于 `config/default.json`，当前生效配置保存在 `%AppData%\WinMouseFix\config.json`。多套配置保存在 `profiles/`，备份保存在 `backups/`，当前配置档案名称记录在 `active-profile.txt`。

完整示例见 [`config/default.json`](../config/default.json)。顶层字段包括 `configVersion`、`enabled`、`pauseInFullscreen`、`receiveBetaUpdates`、`doubleClickSpeed`、`desktopSwipeDirection`、`remaps`、`scroll`、`timing`、`excludedApps` 和 `startup`；`desktopSwipeDirection` 仅为旧配置兼容字段，新版融合动作使用固定方向。

`remaps` 中的每一项包含 `button`、`trigger` 和 `action`。按钮支持 `MButton`、`XButton1` 和 `XButton2`，主要操作为 `click`、`doubleClick`、`hold`、`holdScroll` 和 `holdDrag`；每项动作使用 `{ "type": "Back", "shortcut": "" }` 形式保存。`holdScroll` 只选择一个动作，由该动作内部处理向上和向下；`holdDrag` 同样只选择一个动作，由其内部处理上下左右。旧 `wheelUp`、`wheelDown` 和方向拖动配置继续兼容读取。

按住并滚动的列表显示缩放、音量调节、标签页切换、页面前进后退、虚拟桌面切换、桌面与开始菜单、快速滚动和精确滚动。按住并拖动的列表只显示滚动与移动、虚拟桌面与任务视图、网页与标签导航；普通单次动作不出现在这两个列表中。桌面与开始菜单固定为向上显示桌面、向下打开开始菜单；虚拟桌面与任务视图使用一次 `SendInput` 完成 `Ctrl + Win + Left/Right`，避免桌面切换期间保留组合键状态。

首次启动时，程序读取 Windows 报告的鼠标按键数量。三键鼠标默认使用中键点击、中键长按显示桌面、中键连按打开开始菜单、中键拖动执行虚拟桌面与任务视图；五键或多键鼠标默认使用按键 4 后退、桌面与开始菜单、虚拟桌面与任务视图，以及按键 5 前进、缩放、滚动与移动。无法读取按键数量时使用五键或多键默认值；按钮页的“恢复默认值...”允许手动选择两套默认值。

`scroll` 中保存 `horizontalModifier`、`fastModifier`、`precisionModifier` 和 `zoomModifier`，可选 `none` 或由 `ctrl`、`alt`、`shift`、`win`、`mbutton`、`xbutton1`、`xbutton2` 组成的组合，例如 `ctrl+shift`。默认依次为 `Shift`、`Alt`、`Win`、`Ctrl`；快速滚动使用当前速度的 4 倍，精确滚动使用四分之一，同一组合不能重复分配。`timing.dragThresholdPx` 保存拖动触发距离，范围为 5 至 200 像素，默认 12 像素；鼠标按钮同时设置按住并滚动时，滚轮优先使用修饰键功能。

触摸板保护只针对无活动鼠标按键、无滚轮修饰键的中性滚轮输入：在 300 毫秒内达到 20 次后，核心暂时停止滚轮接管 750 毫秒，让系统恢复原生滚动。Windows 当前事件层无法直接标记设备来源，因此这是连续输入特征判断，不能视为精确的触摸板设备识别。

保存前必须检查字段类型、数值范围和应用路径。配置写入使用临时文件，确认完成后再替换正式文件，避免只保存部分内容；更新程序时保留用户已有设置。

## 9. 设置生效与自动重新载入

按键动作、滚轮参数、总开关和应用排除列表应即时生效。需要重新建立鼠标处理或程序状态的设置，在保存后自动重新启动鼠标功能部分，不关闭可视化界面。

重新启动前保存用户暂停状态和最后一次有效配置，完成后由界面重新发送当前设置。当前页面、窗口位置和大小由界面自身保持；若新配置无法使用，程序恢复上一次有效配置并给出清楚提示。

## 10. 应用排除列表

用户可从正在运行的程序中选择应用，也可手动选择 `.exe` 文件。每条规则支持：

当前版本的应用排除规则采用“全部停用”：命中排除应用时不处理任何增强动作。部分动作停用和按应用使用独立配置暂未加入。

进入排除应用时，程序取消尚未完成的组合动作，并恢复可能仍处于按下状态的按键；离开后自动恢复正常配置。应用识别优先使用完整程序路径，路径不可用时使用程序文件名辅助判断。

## 11. 状态与异常恢复

状态优先级从高到低为：程序退出或重新载入、用户主动暂停、当前应用触发排除规则、正常启用。只有状态真正改变时才更新界面和托盘图标。

全屏暂停只识别处于普通窗口状态且完整覆盖显示器的应用窗口。普通最大化窗口、Windows 桌面、任务栏和虚拟桌面切换界面不触发暂停；桌面切换期间如果鼠标松开事件延迟，核心会在下一次 15ms 状态检查中按物理按键状态复位。

- 配置不存在：识别鼠标按键数量，创建对应默认配置并继续运行。
- 配置内容无效：保留原文件副本，使用默认配置并提示用户。
- 应用路径失效：标记为不可用，但不自动删除规则。
- 动作执行中断：释放程序已按下的按键并清除临时状态。
- 程序再次启动：读取最后一次成功保存的配置。

## 12. 当前限制与后续方向

1. 继续改善连续滚动和触摸板兼容性。
2. 支持更多鼠标按键和多设备独立配置。
3. 评估更精确的输入设备来源识别。
4. 评估 Windows Named Pipes 状态确认机制。

## 13. 验证标准

### 13.1 基础功能

- 运行 `.\scripts\dev.ps1`，界面、鼠标功能和托盘图标正常启动。
- 连续启动程序时只保留一个实例，已有设置窗口被显示到前台。
- 使用 `--background` 启动时窗口不弹出，托盘和鼠标核心正常工作。
- 修改当前设置后，结果自动生效并显示确认状态。
- 在滚动页录入 Ctrl+Shift、Alt+Win 等组合，确认顺序自动规范；输入 A、F1、Ctrl+A 等普通键时保持原值并显示提示。
- 按住完整组合并滚轮时执行对应动作；快速滚动和精确滚动发送普通滚轮，避免部分页面将 Alt/Win 识别为特殊操作。
- 连续使用触摸板滑动页面时不出现频率提示；实体鼠标滚轮、按住滚动和按住拖动仍可用。
- GitHub 更新检查失败时改查 Gitee，Beta 开关开启后可发现预发布版本。
- 退出并重新运行后，所有设置保持不变。
- 暂停和恢复程序时，鼠标原始行为与增强行为正确切换。

### 13.2 应用排除

- 分别测试普通窗口、全屏窗口和无边框全屏窗口。
- 进入排除应用后，侧键、滚轮和组合动作按规则处理。
- 快速切换多个窗口时，状态没有明显延迟或残留。
- 动作进行中切入排除应用时，临时按键状态得到清理。

### 13.3 配置与恢复

- 删除、清空和手动写入无效配置，程序能够恢复并给出提示。
- 修改需要重新载入的设置后，鼠标功能自动重新启动，界面、窗口位置和当前页面保持不变。
- 读取旧版配置时，已有设置能够继续使用。

### 13.4 打包与安装

- `.\scripts\validate.ps1` 执行成功。
- `.\scripts\build.ps1` 生成 `dist/app/WinMouseFix.exe` 和 `dist/app/WinMouseFix.Engine.exe`。
- `.\scripts\package.ps1` 生成 `dist/WinMouseFix-Setup-0.1.4-beta.exe`。
- 安装程序在 Windows 10 22H2 x64 和 Windows 11 x64 中正确检查 .NET Framework 4.8 并启动。
- GUI 主程序及其依赖合计不超过 10 MiB，鼠标核心、配置和资源文件不计入该数值。
- 开始菜单启动、直接更新、卸载、便携运行和登录后静默运行均符合预期。
- 更新或卸载程序时保留用户配置。
