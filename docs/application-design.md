# Win Mouse Fix 应用设计文档

## 1. 文档信息

- 状态：初稿
- 适用版本：首版规划
- 用途：统一产品范围、用户操作、程序组成、发布方式和验收标准

## 2. 产品目标

Win Mouse Fix 是一款基于 AutoHotkey v2 的 Windows 鼠标快捷编辑工具，参考 MacMouseFix 的使用方式，让用户通过可视化页面设置鼠标按键、滚轮和组合动作。

AutoHotkey v2 负责全部核心鼠标功能，GUI 负责把原本需要编写脚本的操作转换为选择项、开关、滑块和表单。目标用户不需要理解 AutoHotkey 语法，只需描述想要的鼠标行为并完成选择。

程序应满足以下要求：

- 用户无需单独安装 AutoHotkey 或其他运行环境。
- 设置完成后立即生效，通常不需要手动重启。
- 指定应用运行时，可暂停部分或全部增强功能，恢复鼠标原始行为。
- 设置在程序关闭、更新或系统重启后仍能保留。
- 程序可常驻系统托盘，并清楚显示当前状态。
- GUI 只提供经过定义的鼠标功能，不要求用户直接编辑脚本。

本产品不是通用 AutoHotkey 编辑器。核心价值是降低鼠标配置难度，同时保留 AutoHotkey v2 在鼠标按键、滚轮、组合动作和应用规则方面的能力。

## 3. 首版范围

首版包含：

- 鼠标侧键和中键的动作设置。
- 滚轮方向与滚动速度设置。
- 常用动作，例如返回、前进、切换窗口和关闭标签页。
- 总开关与系统托盘控制。
- 应用排除列表。
- 设置保存、即时应用和自动重新载入。
- 开机自动运行。
- 配置导入与导出。
- 安装版和便携版发布。

首版暂不包含云端同步、用户账号、在线配置分享、企业集中管理、任意 AutoHotkey 脚本编辑，以及针对每一种鼠标驱动程序的单独适配。完全还原 MacMouseFix 的动画和滚动手感也不作为首版完成标准。

## 4. 用户使用流程

1. 用户安装或直接运行 Win Mouse Fix。
2. 程序显示设置窗口，并在系统托盘中保持运行。
3. 用户设置鼠标按键、滚轮或应用排除规则。
4. 用户修改选项后，程序自动保存配置并立即更新行为。
5. 如果某项设置需要重新初始化，程序只重新启动鼠标功能部分，设置窗口保持打开。
6. 用户切换到排除列表中的应用时，增强功能自动暂停。
7. 用户离开该应用后，原有设置自动恢复。

## 5. 界面设计

主窗口采用左侧导航和右侧设置区域，包含以下页面：

- **概览**：总开关、当前配置、运行状态和最近提示。
- **按键**：侧键、中键及组合动作设置。
- **滚轮**：方向、速度和连续滚动参数。
- **应用**：排除列表及每个应用的处理方式。
- **常规**：开机运行、通知、配置导入导出和版本信息。

界面和托盘图标应区分正常启用、用户暂停、因当前应用暂停、配置重新载入中，以及配置读取失败后使用默认设置。普通选项不设置额外的“应用”按钮，修改后自动保存；需要重新初始化时只重新启动鼠标功能部分。

## 6. 技术与发布方案

### 6.1 产品结构

程序分为“可视化界面”和“AutoHotkey v2 鼠标核心”两部分。鼠标核心是固定基础，负责所有实际鼠标行为；界面负责让用户以简单方式编辑核心所需的配置。

GUI 确定使用 **.NET 8 + WPF**。WPF 负责设置窗口、托盘、状态提示和 Windows 系统功能；`CommunityToolkit.Mvvm` 用于组织页面状态，`System.Text.Json` 用于读写配置。

### 6.2 AutoHotkey v2 鼠标核心

AutoHotkey v2 是本项目确定使用的核心技术，负责鼠标按键、滚轮、组合动作、活动应用识别、应用排除和配置切换。核心读取固定格式的配置并执行预先提供的动作，不要求普通用户编写脚本。

Ahk2Exe 负责把鼠标核心及 AutoHotkey v2 运行环境打包为应用内部程序。用户安装 Win Mouse Fix 后无需单独安装 AutoHotkey。

### 6.3 GUI 封装

GUI 将核心能力显示为容易理解的控件，例如“侧键 1 执行什么动作”“是否反转滚轮”“哪些应用中暂停功能”。用户提交表单后，GUI 检查输入并生成核心能够读取的配置。

GUI 不直接实现鼠标行为，也不向普通用户展示脚本代码。新增鼠标功能时，必须同时增加核心动作、配置定义和对应的 GUI 控件。

### 6.4 两部分通信

界面与鼠标功能通过 Windows Named Pipes 交换结构固定的 JSON 消息。.NET 使用 `NamedPipeClientStream` 连接 AutoHotkey v2 核心，消息包括应用配置、暂停、恢复、重新启动核心、查询状态和执行结果。

GUI 是配置文件的唯一修改者；AutoHotkey v2 只读取经过检查的配置并执行鼠标行为。每次更新都携带配置版本，核心成功应用后返回相同版本；若新配置不能使用，核心继续使用上一份有效配置并把原因返回 GUI。

开关和选项在修改后立即发送，连续滑块在停止变化约 150 毫秒后发送。大多数设置直接更新运行状态，只有不能直接更新的设置才重新启动 AutoHotkey v2 核心，GUI 和托盘始终保持运行。

### 6.5 Inno Setup

Inno Setup 将界面、鼠标功能及所需运行文件组合为统一安装程序，负责安装、卸载、开始菜单和桌面快捷方式、开机运行及版本信息。正式版本同时提供安装版和便携版，生成文件统一放入 `dist/`。

## 7. 程序组成

```text
src/
  gui/
    WinMouseFix.Gui.csproj
    App.xaml
    Views/
    ViewModels/
    Services/
  engine/
    MouseEngine.ahk
    ActionManager.ahk
    AppRuleManager.ahk
    ConfigManager.ahk
    MouseManager.ahk
    StateManager.ahk
  shared/
    config.schema.json
    messages.json
config/
  default.json
assets/
scripts/
tests/manual/
dist/
```

- `gui/`：.NET 8 + WPF 应用，启动可视化页面、管理托盘并展示运行状态。
- `engine/MouseEngine.ahk`：启动 AutoHotkey v2 鼠标核心。
- `MouseManager`：接收并处理鼠标行为。
- `ActionManager`：把用户选择转换为具体动作。
- `AppRuleManager`：识别当前应用并决定是否暂停功能。
- `ConfigManager`：读取、检查、保存和更新配置。
- `StateManager`：管理启用、暂停和重新载入状态。
- `shared/`：规定配置格式以及界面与鼠标功能之间的消息格式。

## 8. 配置设计

默认配置位于 `config/default.json`，用户配置保存在 `%AppData%\WinMouseFix\config.json`。

```json
{
  "configVersion": 1,
  "enabled": true,
  "scroll": {
    "reverse": false,
    "speed": 1.0
  },
  "buttons": {
    "XButton1": "Back",
    "XButton2": "Forward"
  },
  "excludedApps": [
    {
      "path": "C:\\Games\\Example\\Game.exe",
      "mode": "disableAll"
    }
  ],
  "startup": {
    "runAtLogin": false
  }
}
```

保存前必须检查字段类型、数值范围和应用路径。配置写入使用临时文件，确认完成后再替换正式文件，避免只保存部分内容；更新程序时保留用户已有设置。

## 9. 设置生效与自动重新载入

按键动作、滚轮参数、总开关和应用排除列表应即时生效。需要重新建立鼠标处理或程序状态的设置，在保存后自动重新启动鼠标功能部分，不关闭可视化界面。

重新启动前保存用户暂停状态和最后一次有效配置，完成后由界面重新发送当前设置。当前页面、窗口位置和大小由界面自身保持；若新配置无法使用，程序恢复上一次有效配置并给出清楚提示。

## 10. 应用排除列表

用户可从正在运行的程序中选择应用，也可手动选择 `.exe` 文件。每条规则支持：

- **全部停用**：不处理任何增强动作。
- **只停用手势**：保留基础按键设置。
- **使用独立配置**：为该应用指定另一套参数。

进入排除应用时，程序取消尚未完成的组合动作，并恢复可能仍处于按下状态的按键；离开后自动恢复正常配置。应用识别优先使用完整程序路径，路径不可用时使用程序文件名辅助判断。

## 11. 状态与异常恢复

状态优先级从高到低为：程序退出或重新载入、用户主动暂停、当前应用触发排除规则、正常启用。只有状态真正改变时才更新界面和托盘图标。

- 配置不存在：创建默认配置并继续运行。
- 配置内容无效：保留原文件副本，使用默认配置并提示用户。
- 应用路径失效：标记为不可用，但不自动删除规则。
- 动作执行中断：释放程序已按下的按键并清除临时状态。
- 程序再次启动：读取最后一次成功保存的配置。

## 12. 分阶段计划

1. **技术验证**：使用 .NET 8 + WPF 制作最小页面，验证配置保存、Named Pipes 通信、AutoHotkey v2 即时生效、核心重新启动和组合打包。
2. **基础版本**：完成窗口、托盘、配置保存、按键、滚轮和总开关。
3. **应用规则**：完成排除列表、活动窗口识别、状态切换和独立配置。
4. **发布准备**：完成界面与鼠标功能的组合打包、Inno Setup 安装程序、开机运行、配置导入导出和异常恢复。
5. **体验改善**：根据实际使用结果改善连续滚动、组合动作、提示信息和界面细节。

## 13. 验证标准

### 13.1 基础功能

- 运行 `pwsh -File .\scripts\dev.ps1`，界面、鼠标功能和托盘图标正常启动。
- 修改每个首版设置后，结果自动生效并显示确认状态。
- 退出并重新运行后，所有设置保持不变。
- 暂停和恢复程序时，鼠标原始行为与增强行为正确切换。

### 13.2 应用排除

- 分别测试普通窗口、全屏窗口和无边框全屏窗口。
- 进入排除应用后，侧键、滚轮和组合动作按规则处理。
- 快速切换多个窗口时，状态没有明显延迟或残留。
- 动作进行中切入排除应用时，临时按键状态得到清理。

### 13.3 配置与恢复

- 删除、清空和手动写入无效配置，程序能够恢复并给出提示。
- 修改需要重新载入的设置后，鼠标功能自动重新启动，界面、窗口位置和当前页面保持不变。
- 读取旧版配置时，已有设置能够继续使用。

### 13.4 打包与安装

- `pwsh -File .\scripts\validate.ps1` 执行成功。
- `pwsh -File .\scripts\build.ps1` 生成 `dist/WinMouseFix.exe`。
- 打包程序可在没有单独安装项目运行环境的 Windows 电脑中运行。
- 安装、卸载、便携运行和开机运行均符合预期。
- 更新或卸载程序时保留用户配置。
